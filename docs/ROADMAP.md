# Phased Roadmap

## Phase 0: Discovery

Outputs:

- Purified prompt.
- Discovery notes.
- Research baseline.
- Current thinking.

Exit:

- Major risks known.
- No code started.

## Phase 1: Requirements

Outputs:

- Requirement IDs.
- Functional and non-functional requirements.
- Open questions.

Exit:

- Requirements traceable to prompt and discovery.

## Phase 2: Agile SRS

Outputs:

- SRS.
- MVP boundaries.
- Epics and definitions of ready/done.

Exit:

- User approves SRS direction.

## Phase 3: Architecture Validation

Outputs:

- ADRs for mobile, backend, database, maps, AI, auth, storage, moderation.
- Spike plans.
- Cost and risk estimates.

Exit:

- Critical choices have prototypes or reversal conditions.

## Phase 4: UX And Prototype Spikes

Outputs:

- Capture flow wireframes.
- Map prototype.
- Upload prototype.
- AI scoring sample pipeline.

Exit:

- App feel and technical feasibility validated.

## Phase 5: Repo Scaffold And CI

Outputs:

- Flutter app skeleton.
- Backend skeleton.
- Database migrations.
- Docker local stack.
- CI for lint, test, format, docs checks.

Exit:

- New contributors can run the repo locally.

## Phase 6: Auth Foundation

Outputs:

- Google sign-in.
- Email/password.
- Reset flow.
- Account deletion stub.
- Authenticated API.

Exit:

- Secure identity works end to end.

## Phase 7: Capture And Media

Outputs:

- Camera capture.
- Foreground location attach.
- Signed upload.
- Thumbnail generation.
- Private collection draft.

Exit:

- A real animal photo can be uploaded and viewed privately.

## Phase 8: Map Foundation

Outputs:

- Player map.
- Clustered animal activity.
- Privacy-safe regions.
- Waypoint route prototype.

Exit:

- Map supports discovery without exposing exact public points.

## Phase 9: Scoring Prototype

Outputs:

- AI scoring schema.
- Deterministic prechecks.
- Score events.
- Pending/review states.

Exit:

- A submission receives a versioned score.

## Phase 10: Duplicate And Zoo Hardening

Outputs:

- Gold datasets.
- Duplicate hash/embedding tests.
- Zoo geofence import.
- GPS uncertainty rules.

Exit:

- Key anti-farming systems pass benchmark thresholds.

## Phase 11: Collections And Profiles

Outputs:

- User profile.
- Animal collection pages.
- Captive/zoo labels.
- Privacy controls.

Exit:

- Users can manage their identity and captures.

## Phase 12: Social And Groups

Outputs:

- Friends.
- Groups.
- Posts.
- Likes, comments, reposts, hashtags.
- Blocking and reporting.

Exit:

- Social loop works with moderation hooks.

## Phase 13: Leaderboards And Economy

Outputs:

- Global, country, local, friends leaderboards.
- Catch-up mechanics.
- Diminishing returns.
- Fraud-damped social score.

Exit:

- Rankings are fair enough for beta testing.

## Phase 14: Moderation And Admin

Outputs:

- Review queues.
- Admin tools.
- Appeals.
- Policy logs.
- Abuse dashboards.

Exit:

- UGC launch requirements are satisfied.

## Phase 15: Android Alpha APK

Outputs:

- Signed internal APK.
- Test accounts.
- Crash reporting.
- Manual QA scripts.

Exit:

- Closed users can install and test safely.

## Phase 16: Google Play Beta

Outputs:

- Android App Bundle.
- Store listing draft.
- Data Safety form.
- Privacy policy.

Exit:

- Closed/open testing is approved.

## Phase 17: Android Production

Outputs:

- Production release.
- Monitoring.
- Support process.
- Incident response.

Exit:

- Public Android launch.

## Phase 18: iOS Adaptation

Outputs:

- iOS permissions.
- Sign in with Apple.
- TestFlight build.
- App Privacy labels.

Exit:

- TestFlight beta approved.

## Phase 19: App Store Production

Outputs:

- App Store release.
- iOS monitoring.
- Cross-platform parity plan.

Exit:

- Public iOS launch.

# Sprint Plans

Sprint plans turn accepted requirements, ADRs, and work packages into bounded implementation windows.

Rules:

- A sprint plan must name owned files/modules.
- A sprint plan must list forbidden files/modules.
- A sprint plan must include acceptance criteria, tests, privacy/security notes, rollback, and commit sequence.
- Sprint work must use short-burst semantic commits with AI attribution.
- State docs must be updated before sprint close.
